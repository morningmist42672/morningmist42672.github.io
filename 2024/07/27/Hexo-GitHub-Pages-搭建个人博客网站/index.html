6.2.4 总结

以后不管是在 A 还是在 B，亦或是以后还有 C，反正都是

最好先把仓库的最新版本同步到本地（git pull），然后写博客，然后再上传到仓库（git push），最后再发布博客。发布放在最后是一个好习惯，如果先发布博客，但发布后因为电脑突然出问题或者什么别的原因（概率极低，但还是有概率）导致没有上传仓库，那么最新版本的 .md 文件将丢失。先上传仓库则绝对不会有这个问题。
